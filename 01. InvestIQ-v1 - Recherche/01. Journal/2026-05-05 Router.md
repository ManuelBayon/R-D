
Un **router** est la couche qui répond à :

> “Cet event appartient à quel domaine du système ?”

Le **dispatch** répond ensuite à :

> “Quelle fonction précise traite ce type d’event ?”

Donc :

```
event loop = orchestration
router = choix du domaine propriétaire
dispatch = choix du handler exact
handler = transition d’état
```

## Pipeline mental

```
queue  
  ↓
event loop
  ↓
router.route(event)
  ↓
domain module
  ↓
local dispatch
  ↓
handler(event, state)
  ↓
TransitionResult(events, effects)
  ↓
queue.extend(result.events)
```

---
# Ownership des mutations d’état

## Sans router

```
event → handler (global)
```

Problèmes :

- n’importe quel handler peut toucher n’importe quel state
- couplage implicite
- difficile d’auditer

---

## Avec router

```
event → domaine → handler
```

Tu imposes :

```
BarEvent → MarketDomain → peut modifier MarketState uniquementFillEvent → Execution/PortfolioDomain → peut modifier Orders + Portfolio
```

👉 ça devient une contrainte système, pas juste de la lisibilité
