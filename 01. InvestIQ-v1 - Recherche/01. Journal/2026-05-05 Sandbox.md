
Exercice du jour pour apprivoiser l'event-driven.

## Exemple de consigne réaliste

> “Construis un système qui prend :
> 
> - BarEvent
> - produit éventuellement un DecisionEvent
> - puis un FillEvent
> - et met à jour une position
> 
> Je veux que tu puisses rejouer 10 events et obtenir le même résultat.”

Rien de plus.

## Garde uniquement

```
BarEvent → DecisionEvent → FillEvent → Position
```

Un seul state :

```
Position:    
	qty    
	avg_price
```

Un seul invariant :

```
qty == 0 ⇒ avg_price = 0 (ou None)
```

## Et une seule boucle

```
for event in events:    
	
	state = transition(state, event)
```

## Critère de réussite

Tu dois pouvoir :

```
1. écrire une séquence d’events à la main2. prédire l’état final3. vérifier que le code donne exactement ça
```
